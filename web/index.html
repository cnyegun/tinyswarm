    How it works

    Step 1

    Scan

    Automated tools (axe-core) inspect the page for known WCAG 2.2 violations and structural issues.

    Step 2

    Rewrite

    Agents iteratively rewrite the markup, addressing each finding while preserving meaning and visual intent.

    Step 3

    Review

    A panel of reviewer agents vote on the rewrite. Only an accepted version is published as the improved page.

    Methodology

    Each audit runs in isolation. The page is fetched once and all transformations are performed against a local copy.

    Results are supporting evidence of WCAG 2.2, it is not a WCAG conformance statement. Test and verify with assistive-technology before relying on them.

    AAA — Accessibility AI Audit

    AAA runs an accessibility swarm against a web page: it captures the source, asks agents to rewrite it, checks the result with Playwright and axe, and serves the transformed page and report.
